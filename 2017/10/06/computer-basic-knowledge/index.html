搞定计算机基础

发表于 2017-10-06 | 分类于 计算机基础

前言

私认为计算机基础分为6大块：数据结构与算法，操作系统，离散数学，编译原理，数据库原理和计算机网络。

数据结构与算法

数据结构包括栈，队列，链表，树，图。

算法有动态规划算法，贪心算法， 排序算法， 搜索算法 ### 栈和队列

  • Stack是一种LIFO(Last In First Out)的结构

    Stack的应用范围非常广，比如JVM就是基于Stack模型的机器

  • Queue是一种LILO(Last In Last Out)的结构

链表

  • 单向链接

    一般用next连接下一个结点

  • 双向链表

    一般用next连接下一个结点，用previous连接上一个结点

  • 单向循环链表

    链表的最后一个结点的next连接链表的第一个结点

  • 双向循环链表

    链表的最后一个结点的next连接链表的第一个结点，第一个结点的previous连接最后一个节点

树

  • 二叉搜索树

    二叉搜索树insert，delete，search的复杂度为\(O(logn)\)

  • 红黑树

    前面说了二叉搜索树的复杂度为\(0(logn)\),但实际中可能恶化成\(O(n)\),比如所有的元素都是有序插入的.因此引出了平衡二叉树。平衡二叉树尽力使得树的高度更小，故会尽可能充分的利用左右结点，会造成左子树与右子树高度之差不会超过1。

在数学上，图表示物体与物体之间的关系。图由顶点与边构成。我们根据边的有无权重, 有无方向,分为以下情况：

    • BFS

      Breadth-First-Search.宽度优先搜索

      一层一层的搜索 ，利用Queue缓存顶点实现。

  • 无向加权图

  • 有向图

  • 有向加权图

    • Shortest Path

      • Dijkstra Algorithm\((o(n^2))\)

        主要解决单点s最短路径问题。2个数组:距离数组d,初始化为无穷大, 前驱数组previous，初始化为null.将s的邻结点加入queue中，并设置好2个数组中这些顶点的值。之后从queue取出顶点x，看x的邻结点的是否有更短的路径\(d[z] > d[x] + w(x, z)\),有则更新previous和d.重复此过程直到queue为空。

    • Minimum Spanning Tree

      • KruskalMST\((O(ElogE))\)

        将所有的边放入一个优先队列中，依次拿出最小权重的一条。如果该边的2个端点不在MST中，则往MST加入这条边，直到所有的结点都在MST中。

      • PrimMST\((O(Elog|V|))\)

        从某一个结点X开始，将该节点领接边加入一个优先队列中，选择队列中最小权重的一条边，如果该边的2个端点不在MST中，则往MST加入这条边，让这条边的末端结点成为X，循环上诉的步骤直到所有的结点都在MST中。

排序算法

  • 快速排序(不稳定 \(O(nlogn)\))

    主要的思想是划分，将数组划分为2部分，小于等于分界点和大于分界点的。一直递归的划分下去，直到不用划分。

  • 堆排序(不稳定 \(O(nlogn)\))

    ​将待排序数组建堆，然后每次将顶点与末尾结点互换，将堆大小减一，然后重新调整堆（sink），重复。

  • 归并排序(稳定 \(O(nlogn)\))

    主要思想是将2个已排序好的数组归并排序成一个排序数组。先从2个只包含一个元素的数组开始归并，然后递归的进行。

  • 希尔排序(不稳定 \(O(n^{\frac{2}{3}})\)或者\(O(nlog^2n)\))

    感觉叫间隔排序。先排序按一定间隔的数组排序，然后将间隔变小，继续排序。知道间隔为1.间隔选择不同，复杂度也不一样。

  • 选择排序(稳定 \(O(n^2)\))

    ​待排序集合\(E\), 选出最大值\(A\), \(E \to {E - A}\),再选择最大值\(A\),重复知道\(E \to 0\)。

  • 插入排序(稳定 \(O(n^2)\))

    将元素一个个从头插入

  • 冒泡排序(稳定 \(O(n^2)\))

    和冒泡类似，每次循环把最大的值冒到最后。

进程和线程

进程是计算机的一个基本单位。进程包括进程块，程序。在内核中有进程调度表来调度进程。

多进程通信方式：

  • 管道

  • 信号(signal)

  • 消息队列

    进程发送消息，另一个进程接收线性。系统级实现。

  • 共享内存

  • 信号量(semaphore)

    主要用于进程间同步

  • 套接字(socket)

    可用于本机和跨主机进程通信

线程是进程的一个基本单位。线程共享进程内的数据。这就引发一些竞争状态。

我们通常会利用锁进制来处理并发冲突：

  • 悲观锁

    每次获取数据的时候，都会担心数据被修改，所以每次获取数据的时候都会进行加锁，确保在自己使用的过程中数据不会被别人修改，使用完成后进行数据解锁。由于数据进行加锁，期间对该数据进行读写的其他线程都会进行等待。

    悲观锁：比较适合写入操作比较频繁的场景，如果出现大量的读取操作，每次读取的时候都会进行加锁，这样会增加大量的锁的开销，降低了系统的吞吐量。

  • 乐观锁

    每次获取数据的时候，都不会担心数据被修改，所以每次获取数据的时候都不会进行加锁，但是在更新数据的时候需要判断该数据是否被别人修改过。如果数据被其他线程修改，则不进行数据更新，如果数据没有被其他线程修改，则进行数据更新。由于数据没有进行加锁，期间该数据可以被其他线程进行读写操作。

    乐观锁：比较适合读取操作比较频繁的场景，如果出现大量的写入操作，数据发生冲突的可能性就会增大，为了保证数据的一致性，应用层需要不断的重新获取数据，这样会增加大量的查询操作，降低了系统的吞吐量。

    例子：CAS(compare and set),一般会使用值value和版本version，来避免ABA问题。

    总结：两种所各有优缺点，读取频繁使用乐观锁，写入频繁使用悲观锁。

进程和线程都有几率造成死锁。死锁是指多个进程/线程循环等待彼此占有的资源而无限期的僵持等待下去的局面。

虚拟存储器

虚拟存储器为进程提供了一个假象，仿佛进程使用的是一个连续无限的地址空间（虚拟存储器的关键就是对地址空间的管理），所谓的磁盘空间换内存只是虚拟存储器的一个结果，而并非目的。

页表，是管理页的（内存以页为单位，页大小一般为4KB）。一般用多级页表来管理（节省内存）。

虚拟存储器将虚拟地址转为物理地址。虚拟存储器系统拿到虚拟地址，查找页表，找到页表中一项。如果命中，则说明相应的物理页是缓存在内存中的，否则就触发缺页异常。

TCP

  • 三次握手

    有2个主机client和server，它们要建立TCP连接，需要经过以下3个步骤：

    1. client向server发送一个SYN来创建一个主动打开，这个SYN的序号seq设定为随机数R
    2. server回送一个SYN/ACK.ACK=R+1, SYN的seq设定为随机数X
    3. client回送一个ACK，ACK=X+1

  • 四次挥手

    1. client发送FIN请求\(FIN=1, seq = u\), 进入FIN-WAIT-1阶段
    2. server收到请求，发送ACK\(ACK=1, seq = v, ack = u+1\),server进入CLOSE-WAIT，client进入FIN-WAIT-2
    3. server将剩余的数据传送给客户端后，发送FIN请求\(FIN=1, ACK=1, seq = w, ack =u+1\)
    4. client收到FIN请求，发送ACK确认\(ACK=1,seq=u+1,ack=w+1\).server收到请求关闭，client在2MSL后关闭。

  • 数据传输

    TCP协议是以包(packet)为单元传送数据的。

    TCP发送方会为每个包设定一个序列号X(seq),当接收方接收了这个包，会发送一个ACK(seq = X+packet.length)请求。下一个包的seq也是这个编号(X+packet.length)。

    • 超时重传

      如果发送方没有收到确认请求，发送方会重传数据。

    • 乱序恢复

      假如我们发送了2,3,4个packet过去。3号packet丢失了。那么接收方的ACK会一直为3(ACK的编号总是Max(已接收packet编号)+1),即使收到了4号packet，ACK的编号也不会变。这样发送方就知道3号丢失，然后重新发送3号。只重传3号或重传3号之后所有的就是看采用的TCP的重传策略了。

  • 流量控制

    TCP使用滑动窗口机制来进行流量控制。

    每一个TCP方都有一个发送窗口和接收窗口。发送窗口的开端总是在已接收ACK编号的位置。当ACK的编号移动时，它就会滑动。接收窗口的大小限定了它发送的数量。

    接收窗口限定了此刻接收方还能接收的数量。接收窗口的开端总是在已接收数据+1的位置。

  • 拥塞处理

    • 慢开始

      在最开始的时候，设CWND(congestion window)为一个MSS(Maximum Segment Size).在每一个传输回合后，大小*2，直到阀值(ssthresh)

    • 拥塞避免

      让CWND缓慢增大，每一次来回增加1.当超过慢开始阀值(ssthresh),开始进行拥塞避免算法。2种情况：

      1. 乘法减少.不论是慢开始还是拥塞避免阶段，只要出现超时重传的情况，就将ssthresh减半
      2. 加分增大。执行拥塞避免算法后，为避免过早出现拥塞，是CWND缓慢增大

    • 快重传算法

      发送方只要连续收到3个重复确认就应重传对方尚未收到的数据包，而不必等到超时。

    • 快恢复算法

      • 当发送方收到3个连续的重复确认。开始实行乘法减少算法。但不执行慢开始算法
      • 乐观估计网络并没有发送拥塞，实行加分增大算法，是CWND线性缓慢增长。

SQL

  • 事务
    • ACID
      • 原子性(Atomicity)
      • 一致性(Consistency)
      • 隔离性(Isolation)
      • 持久性(Durability)
    • 四种隔离级别
      • 串行化
      • 可重复读：事务A，B并行。如果另外一个事务B新增，删除了数据，事务A看不到这种变化。比如user表有9条数据，事务B删除了一条数据，事务A执行select * from user 还是会得到结果9.这就是幻读
      • 读取已提交：事务B删除/修改了一条数据并提交了，事务A可见，能看到这种变化。
      • 读取未提交：事务B删除/修改了一条数据(不用提交)，事务A可见，能看到这种变化.如果事务B回滚，事务A读到的数据就是无效的，这就是脏读。
