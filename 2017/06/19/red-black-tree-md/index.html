RED-BLACK-TREE(红黑树)

发表于 2017-06-19 | 分类于 数据结构与算法

简介

红黑树是一种非常常见的平衡树，比如Java8的HashMap中就使用了。

下面我主要按照算法导论来学习红黑树， 用C++实现。

如其名，红黑树的结点有2种颜色–红/黑。 我们先定义一下树结点：

接下来看看红黑树的性质: 1. 每个结点是红色的，或是黑色的

  1. 根结点是黑色的

  2. 每个叶结点是黑色的

  3. 若一个结点是红色的， 则它的2个结点都是黑色的

  4. 对每个结点， 从该结点到其所有后代叶节点的简单路径上， 均包含相同数目的黑色结点

旋转

当对树进行insert或delete操作后，可能会破坏树的性质。这时，我们需要使用旋转来进行修复。旋转分为左旋和右旋，这2个操作是镜像的。 让我们先看看左旋： 假设要旋转的结点为x, y为x的右孩子（y不可为null）。暂时没图，先上代码了 如果y为null，啥都不干，直接返回。否则让y成为该子树的根结点， x成为y的左孩子， y的左孩子成为x的右孩子。

插入

插入操作与普通的二叉树插入类似，不同之处在于设置结点的颜色，和一个修复操作。

由于我们插入的结点x颜色为红色，可能违背了树的一或多条性质，故为了平衡，我们需要进行修复。 如果x的父节点颜色不是红色，那么就没有违背红黑树的性质，因此不需要进行任何操作。 不走运的话,x的父节点是红色的。那么现在就有3种情况进行讨论(假设x的父节点是左孩子，x的父节点是右孩子的操作是镜像的) case 1:uncle 节点是红色 case 2: uncle 节点是黑色，且当前节点是右孩子 case 3: uncle 节点是黑色， 当前节点是左孩子 让我们在代码里看看：

删除

红黑树删除结点步骤和二叉树删除操作类似,不过多了检测操作,判断需不需要进行修复操作. 我们需要记录fixNode和fixNode的parent(因为没有使用哨兵结点,之后的修复操作,当fixNode为空的时候,需要进行记录这个) 先看左结点为空的情况,这是fixNode就为被删除结点(x)的右结点,fixNode_parent就是x的parent,然后判断fixNode是左右结点,originColor记录x的颜色. 再来看看左右结点都存在的情况.这里的策略是:选择x的右子树最小的结点z.fixNode就是z的右结点,fixNode的parent就是z的parent.originColor记录z的颜色. 然后将z替带x,并将x的颜色赋给z.如果x是z的父节点,则fixNode_parent修正为z.

如果originColor为黑色,则需要进行修复操作.

接下来看看删除修复操作. 删除的修复有4种情况: 1. x的兄弟结点w是红色的 2. x的兄弟结点w是黑色的, 而且w的2个子结点都是黑色的 3. x的兄弟结点w是黑色的, w的左孩子是红色的, w的右孩子是黑色的 4. x的兄弟结点w是黑色的, w的右孩子是红色的

比较注意的是:x,故在第一次调用的时候,把x_parent,和是否左孩子都传递过来了.所以在方法中需要注意x_parent的赋值.
