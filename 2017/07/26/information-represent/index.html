信息的表示和处理

发表于 2017-07-26 | 分类于 CSAPP笔记

1.整数表示

本文约定,默认是二进制编码表示, 表现形式为X的向量组\([x_{w-1} x_{w-2} ... x_0]\) , w默认为32.

  • 无符号数

    无符号数指大于等于0的整数.比如10表示为

    \([00000000000000000000000000001010]\)

    则计算方式为 \[ Y = \sum_{i=0}^{w-1} x_i 2^{i} \]

  • 有符号数

    有符号数能表示负数和非负数,一般采用补码(two’s-complement),比如-10表示为

    \[[11111111111111111111111111110110]\]

    则计算方式为 \[ Y = -x_{w-1} 2^{w-1} + \sum_{i=0}^{w-2} x_i 2^{i} \]

  • 相互转换

    无符号数与有符号树相互转换,位不变,只是改变了解释的方式.

  • 扩展

    为了简单,我们假设将w=8,扩展到16,符号\(\cdot\)之前的是扩展的位

    • 无符号数扩展 \[ [x_{w-1}x_{w-2}...x_0] \to^{extend} [00000000 \cdot x_{w-1}x_{w-2}...x_0] \]

    • 有符号数扩展 \[ [x_{w-1}x_{w-2}...x_0] \to^{extend} [x_{w-1}x_{w-1}x_{w-1}x_{w-1}x_{w-1}x_{w-1}x_{w-1}x_{w-1}\cdot x_{w-1}x_{w-2}...x_0] \]

  • 截断

    截断和扩展相反.设K为截断的位.无符号数和有符号数都是相同的操作,截断后的数按照之前的数表示方式来解释. \[ [x_{k-1}x_{k-1}x_{k-2}x_{k-3}\dots x_{0}\cdot x_{w-1}x_{w-2}...x_0]\to^{trunc} [x_{w-1}x_{w-2}...x_0] \]

2. 整数运算

  • 无符号加法

    设无符号数\(u_1\)和\(u_2\),则 \[ u_1 + u_2 = (u_1 + u_2) \mod 2^w \]

  • 有符号加法

    设有符号数\(s_1\)和\(s_2\),\(U2S\)是无符号转成有符号,\(S2U\)是有符号转成无符号,则 \[ s_1 + s_2 = U2S((S2U(s_1)+S2U(s_2))\mod 2^w) \]

  • 补码的非

    对于补码\(x\),则补码的非为 \[ -x = \begin{cases} -2^{w-1} ,&\quad x = -2^{w-1} \\ -x, &\quad x > -2^{w-1} \end{cases} \]

  • 无符号乘法

    设无符号数\(u_1\)和\(u_2\),则 \[ u_1 \cdot u_2 = (u_1 \cdot u_2) \mod 2^w \]

  • 补码乘法 \[ s_1 \cdot s_2 = U2S((s_1 \cdot s_2) \mod 2^w) \]

  • 乘以常数

    在多数机器上,整数乘法指令相当慢,需要10个或更多时钟周期,其他整数运算指令(如加法,减法,移位)只需要1个时钟周期.因为,编译器会使用一种优化,试着用移位和加法操作的组合来代替乘以常数因子的乘法.

    更具象一点,举一个例子\(x \cdot 14(2^3 + 2^2 + 2^1)\), 则可能会优化成\((x << 3) + (x << 2) + (x << 1)\)

  • 除以2的幂

    在多数机器上,整数除法指令比乘法指令更慢,故编译器可以用右移操作来优化一些除法操作.

    • 无符号数

      采用逻辑右移操作 \[ x / 2^k = x >> k \]

    • 有符号数

      采用算术右移操作,但是计算机除法会造成舍入操作,比如会把-5/2舍入为-3,结果应该是-2.为处理这种情况,我们需要偏置这个除数.

      原理:\(\lceil x/y \rceil = \lfloor (x+y-1)/y\rfloor\) ,故整个公式为 \[ x/2^k = \begin{cases} x >>> k,&\quad x >= 0 \\ (x + (1 << k) + 1) >>> k, &\quad x < 0 \end{cases} \]

3. 浮点数

  • 浮点数表示

    IEEE浮点标准用\(V= (-1)^s \times M \times 2^E\) 的形式来表示一个数.

    • 符号(sign) \(s\) 决定这个数是负数\((s=1)\)还是正数\((s=0)\),对于数值0的符号位解释视为特殊情况

    • 尾数\(M\) 是一个二进制小数,它的范围是\(1 \sim 2-\varepsilon\) 或者是\(0 \sim 1-\varepsilon\)

    • 阶码\(E\)的作用是对浮点数加权,这个权重是2的\(E\)次幂

      将浮点数的位表示划分为三个字段,分别对这些值进行编码:

    • 一个单独的符号位\(s\)直接编码符号\(s\)

    • k位的阶码字段\(exp=e_{k-1}\dots e_1 e_0\)编码阶段\(E\)

    • n位小数字段\(frac=f_{n-1} \dots f_1 f_0\) 编码尾数\(M\),但是编码出来的值也依赖阶码字段的值是否等于0

      下面给出float(32 bit)和double(64 bit)的简陋示意图

      float:

      s(31) exp(30:23) frac(22:0)
      \(s = 1\) \(k = 8\) \(n = 23\)

      double:

      s(63) exp(62:52) frac(51:0)
      \(s = 1\) \(k = 11\) \(n = 52\)

    根据exp的值,被编码的值可以分为三种不同的情况(最后一种情况有2个变种),以float为例:

    1. 规格化的,位表示如下\([s|\ne 0 \& \ne 255|frac]\)

      这种情况下,阶码字段被解释为偏置(biased)形式表示的有符号数整数.也就是说\(E = e - Bias\), 其中\(e\)是无符号数,就是上文提到的\(exp\), \(Bias\)是一个等于\(2^{k-1}-1\)的偏置值.对小数字段\(frac\)的解释为描述小数值\(f\),其中\(0 \le f \lt 1\),其二进制表示为\([0.f_{n-1}\cdots f_1 f_2]\).尾数定义为\(M = 1 + f\).

    2. 非规格化的.位表示为\([s|00000000|frac]\)

      在这种情况下, 阶码值是\(E = 1 - Bias\), 而尾数的值是\(M = f\)

    3. 特殊值

      1. 无穷大,位表示为\([s|11111111|00000000000000000000000]\)
      2. NaN,位表示为\([s|00000000|\ne 0]\)

4.浮点数舍入

​ 浮点数默认舍入方式是向偶数舍入
