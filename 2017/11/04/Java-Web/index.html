基本

  • 类型

    byte, short, int,long,float,double,boolean, char,引用类型(包括Array,Class)

  • 变量与常量

    • 变量声明 type name [= express]
    • 常量声明 final type name = constValue;

  • 赋值

    variable = value

  • 条件

  • 循环

    • for loop

    • while loop

  • 基本操作

    • 二元操作

    • 一元操作

      ​

  • package & import

    每一个类都要放在package中。

    可以通过import来引用其他package的类或直接引用类的静态方法(import static)

  • 类

    • (instantiation)实例化

      为类创建一个实例

    • field(字段)

      类信息的载体

    • method(方法)

      类中的操作都封装在方法中。

    • static

      static的field和method可以直接通过(类的名字和类实例访问)。non-static的field和method只能通过类实例访问。

    • Permission Control

      权限控制应用于field和method和inner class,有4种级别:

      • public

        任何类都能访问

      • protect

        同一package的类和子类可以访问

      • default

        同一package的类可以访问

      • private

        只有自己能访问

    • inner class

      写在类中的一个类。

    • inheritance

      继承的优势:

      • code reuse(代码重用)
      • override(重载行为)
      • Specialization(特化)

      劣势:

      • 子类代码依赖父类代码
      • 扩展性差

    • 抽象类(abstract class)

      特殊的类，无法实例化。有自己的field和method。

      • abstract method

        抽象方法一定要被子类重载

      • non-abstract method(default)

        子类可选择性重载

  • interface(接口)

    和abstract class类似，不同的是，接口没有field，接口的方法都是abstract method

IO

  • BIO

    • 字节流 & 字符流

      InputStream是字节流，Reader是字符流

    • 输入流 & 输出流

      InputStream&Reader是输入流， outputStream&Writer是输出流

    • 节点流 & 处理流

      处理节点的流，比如文件是一个结点，网络主机是一个结点。FileInputStream&FileOutputStream就是结点流。

      在流上进行操作的流就是处理流，比如BufferInputStream对inputStream做缓冲操作，DataInputStream对inputStream做一些高级的操作，如readInt。

  • NIO

    待续

反射

反射可以在运行时获得类,接口,字段,方法的信息,可以实例化对象,调用方法,获取和设置field的值.

  • reflect-Class

    我们可以获得对象的Class对象,Class基本包含了所有信息,Class Name, Modifiers,Package Info, SuperClass,

    Implemented Interfaces, Constructors, Fields, Methods, Annotations.

  • Constructors

    我们可以通过Class对象来获得构造函数,得到构造函数的参数,通过构造函数来实例化对象.

  • Field

    通过Class对象来获得所有的fields和某个特定的field,取值或赋值.如果一个field是private的,我们可以让它可见.

  • Method

    通过Class对象获得Method对象, 可以获得Method对象的返回类型和参数,而且还可以调用.

  • Annotation

    通过反射，我们可以拿到class,field, method, parameters的注解信息。

  • Generics

    大多数情况下，泛型信息会在编译期擦除。但有3种情况下，可以通过反射获取具体的泛型信息。

    • method return types

    • method parameter types

    • field types

  • Arrays

    通过反射，我们可以创建数组，访问数组，获得数组元素的类型。

  • Dynamic Proxies

    动态代理可以实现一些类似aop的效果。

并发

  • 创建线程

    2种方式,继承Thread类;实现Runnable接口

  • Atomic

    Java种内置了一些原子类,它们的操作为原子操作.比如AtomicInteger, AtomicBoolean, AtomicLong.

  • ThreadPool

    线程池可以重用已分配的线程,减少分配线程的次数,节省资源和时间.

  • FutureTask

    我们可以利用FutureTask来多一些计算,然后获取计算后的结果并处理.实现FutureTask需要实现Callable<T>接口

  • Compare And Swap(CAS)

    一种无锁设置值算法.

  • wait/notify&notifyAll

    等待/唤醒是线程间通信的一种方式.当一个线程调用任意一个对象X的wait方法,这个线程会被挂起,直到相同对象X的notify(notify随机唤醒一个正在等待该对象的线程)或notifyAll(notify唤醒所有正在等待该对象的线程)方法.

  • volatile

    volatile变量的所有变化都是可见的.对于volatile变量,所有线程访问得到的都是最新的值.

  • synchronizer

    在多线程编程中, 同步应该是最常用的.同步有3种形式:

  • BlockingQueue

    可以用BlockingQueue实现经典的生产者-消费者模式

  • ThreadLocal

    ThreadLocal可以用每个线程持有一个自己的变量,对ThreadLocal持有变量的修改只在本线程可见.

  • Semaphore(信号量)

    信号量经常用了限制可访问临界资源的线程数.当信号量的数量上限设置为1时,可当做锁来使用.

      在上面的例子中,我们需要手动配置依赖,autowired能隐式的注入依赖,autowired内部用的setter注入或constructor注入.以下是配置

  • Spring AOP

    切面编程,通常可用于日志记录,数据库事物管理等工作.

    • Join point

      程序中的一个点(位置),比如函数调用,异常处理等等.Spring AOP仅支持函数调用这个join point.

    • Advice

      在Join point执行的动作.有下面5种.

      • Before Advice

        在join point之前执行

      • After Advice

        在join point之后执行,无论是正常返回或异常

      • After Returning Advice

        在join point正常完成后调用

      • After Throwing Advice

        在join point异常退出后执行

      • Around Advice

        在join point之前和之后执行.

    • Pointcut

      一个表达式语言用来匹配join point的.

    • Introduce

      引入新接口到任意的Advice对象.

    • Target Object

      被应用Advice的对象.

    • Aspect

      包含Advices, Join points, pointcut的类.

    • Interceptor

      只包含一个advice的aspect

    • AOP Proxy

      用来实现AOP的方式,通常有JDK dynamic proxy 和CGLIB proxy.

    • Weaving

      将aspect和其他对象连接生成advised对象的过程.在Spring AOP中,Weaving在运行期间执行.

  • Spring MVC

    一图镇楼

    • DispatcherServlet

      请求分发器.根据请求找到对应的Handler Mapping.

    • HandlerMapping

      找到对应的controller

    • Controller

      处理业务逻辑,返回ModelAndView

    • ModelAndView

      模型和视图.

    • ViewResolver

      视图解析,根据返回的ModelAndView找到指定的视图.

    • View

      视图,负责页面呈现

    常用注解

    以下注解需要在配置中开启注解扫描.

    • @Controller

      指明这个一个控制器,注册到Spring上下文中.

    • @RequestMapping

      设置匹配路径

    • @RequestBody

      该注解用于读取Request请求的body部分数据,经过Spring默认的转换器绑定到方法的参数上

    • @ResponseBody

      返回的内容写到response的body中.

    • @RequestParam

      从请求中获得特定名称的参数,并绑定到对应的方法参数上.

    • @Service

      注册一个bean到Spring上下文中.

    • @Autowired

      自动注入依赖.
