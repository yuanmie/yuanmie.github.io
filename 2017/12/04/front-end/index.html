React

  • JSX

    JSX是JavaScript的一种扩展。

    简单的看，可以在JavaScript中混入HTML

  • Component react以组件为基础

  • state 组件内部状态，可变。通过this.state.xxx获取，this.setState({key:value})设置。

    • lifecycle

  • event handle react包装了原先DOM的事件，符合W3C标准，不担心兼容问题。

  • top-down data flow & inverse data flow

    props会从高层级自动流传到低等级的组件。如果低等级的组件想要改变高等级的组件，需要在props传入高等级组件的方法。

  • key & list

    react中的list中每一项需要一个唯一的key值。

  • refs & findDOMNode

    在一些情况下（比如获取输入），需要得到真实的DOM node（react 使用的是virtual DOM）。

  • Forms

    处理表单时，用state存储所有输入的值，自定义onSubmit事件。

  • Composition first versus inheritance

    强烈推荐使用组合。唯一的继承是extends component

  • Router

    ​包裹多个Route

  • Link Link跳转到指定的Route中。

  • HashRouter 服务静态内容。

Redux

redux主要是为了解决组件间状态交互的问题

  • store 一个仓库,存储state
  • state 状态
  • action 事件，格式为{type:“xxx”, others}
  • dispatch 分发器。把action发送给store
  • reducer 根据收到的action，旧的state，计算出新的state
  • subscribe 订阅，state变化后，订阅的UI组件进行重新渲染。
  • connect 用于从UI组件(纯UI，数据只从props获得)生成容器组件(管理状态)。
    • mapStateToProps() 将容器组件的state转化为UI组件的props
    • mapDispatchToProps() 将UI组件的action传给store
  • privoder 包裹组件，利用react的context机制，使得同一provider下的组件可以方便的访问同一state.

React-Native

用react js写移动端，react-native会把js代码转换成原生代码。

可以把react写web的经验迁移过来，有几个不同点：

  1. 组件

    react native有自己的组件

  2. 布局

    react native只能使用flex布局

  3. 样式

    react native没有全部的css样式，且名称变化如margin-left（css）为marginLeft（react native）。

    样式可以在组件中直接写样式,或者用stylesheet.
