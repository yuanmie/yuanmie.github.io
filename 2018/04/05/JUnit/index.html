发表于 2018-04-05 | 分类于 Java

前言

JUnit 是被广泛使用的测试框架，主要用于单元测试。

在计算机编程中，单元测试（英语：Unit Testing）又称为模块测试, 是针对程序模块（软件设计的最小单位）来进行正确性检验的测试工作。程序单元是应用的最小可测试部件。在过程化编程中，一个单元就是单个程序、函数、过程等；对于面向对象编程，最小单元就是方法，包括基类（超类）、抽象类、或者派生类（子类）中的方法。

https://www.wikiwand.com/zh-hans/%E5%8D%95%E5%85%83%E6%B5%8B%E8%AF%95

使用 JUnit 的好处主要有那么几点：

  • 可以使测试代码与产品代码分开。
  • 针对某一个类的测试代码通过较少的改动便可以应用于另一个类的测试。
  • JUnit 是公开源代码的，可以进行二次开发。
  • 可以方便地对 JUnit 进行扩展。

注解

注解 说明（不特殊说明，默认在方法上注解）
@Test 用于将方法标记为测试方法
@BeforeAll 在所有测试方法之前执行，只会执行一次，必须为静态方法
@AfterAll 在所有测试方法之后执行，只会执行一次，必须为静态方法
@BeforeEach 在每个测试方法之前执行
@AfterEach 在每个测试方法之后执行
@Disable 用于类或方法,禁用测试类或测试方法
@DisplayName 用于类或方法,为测试类或测试方法提供自定义名称
@Tag 给测试方法打上标签，之后可以通过tag来过滤
@IncludeTags 用于类，与@Tag配合，指定Tag的测试方法才会执行
@ExcludeTags 用于类，与@Tag配合，指定Tag的测试方法会被排除，不执行
@RepeatedTest 重复执行测试

有些情况下，我们需要判断程序满足一定的条件下，才运行测试。这就是假设的使用场景了。如果不满足假设，则测试失败。有请官方demo。
